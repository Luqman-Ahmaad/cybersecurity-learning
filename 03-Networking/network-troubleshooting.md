# 🛠️ Network Troubleshooting

## 🎯 Objective
To learn how to identify and fix common network issues using basic tools and techniques.

---

## 🧠 What is Network Troubleshooting?

It is the process of diagnosing and resolving network problems.

---

## 🔍 Common Network Issues

- No internet connection  
- Unable to reach target system  
- DNS not resolving  
- Slow network  

---

## 🧪 Essential Troubleshooting Commands

### 1️⃣ Check IP Address

ip a


---

### 2️⃣ Test Connectivity

ping google.com


---

### 3️⃣ Trace Route

traceroute google.com


---

### 4️⃣ Check DNS Resolution

nslookup google.com


---

### 5️⃣ View Network Connections

netstat -tulnp


---

## 🔄 Troubleshooting Steps

1. Check physical connection  
2. Verify IP address  
3. Test connectivity (ping)  
4. Check DNS  
5. Analyze routing  

---

## 📡 Example Scenario

### Problem:
Cannot access a website

### Steps:
- Ping website → fails  
- Ping IP → works  
👉 DNS issue  

---

## 🔐 Importance in Cybersecurity

- Helps detect network issues  
- Useful during penetration testing  
- Identifies blocked ports/services  

---

## 🧠 What I Learned

- Troubleshooting is systematic  
- Commands help diagnose issues quickly  
- Understanding networks improves problem-solving  

---

## ⚠️ Common Mistakes

- Skipping basic checks  
- Not verifying IP configuration  
- Ignoring DNS issues
